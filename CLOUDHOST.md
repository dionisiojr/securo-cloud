# 🚀 Securo - Free Cloud Hosting (Render + Supabase)

This repository is a fork optimized so you can host **Securo** in the cloud **100% for free**, without needing to clone the repository or configure a local Docker environment. You just need to connect this link directly to your Render dashboard.

When the application starts for the first time, the system itself will guide you through the initial setup (language, preferences, etc.).

---
## 📂 Files Changed / Added in this Fork

To enable free cloud hosting and resolve environment discrepancies, we modified the minimum number of files possible relative to the original project:
1. `structure.sql` *(Added)*: A consolidated script containing the complete database structure and necessary table fixes.
2. `frontend/default.conf.template` *(Modified)*: Reverted the local Docker DNS resolver (`127.0.0.11`) back to a direct `proxy_pass ${BACKEND_URL};` structure, ensuring Nginx can correctly resolve the public Render backend URL instead of looking for local Docker internal daemons.

## 📋 Prerequisites and Step-by-Step Guide

### 1. Get Pluggy Credentials (Open Finance)
Securo uses the Pluggy API to automatically fetch your banking data.
1. Go to the [Pluggy](https://pluggy.ai/) website and create a free account.
2. In their dashboard, create a **Development Application** (Sandbox/Development).
3. Save the generated **Client ID** and **Client Secret**. They will be used later in Render.
> **Note:** You do not need to configure any Webhook URL in the Pluggy dashboard. Securo uses an *Active Synchronization* model (the update button in the app UI fetches data actively on demand).

---

### 2. Configure the Database (Supabase)
Supabase will act as your cloud PostgreSQL server to store your transactions securely.
1. Create a free account on [Supabase](https://supabase.com/) and start a new project.
2. In the left sidebar, click on **SQL Editor** and then click **New Query**.
3. Open the [`structure.sql`](./structure.sql) file from this repository and copy all its text content.
4. Paste the code into the Supabase editor and click **Run** (in the bottom right corner).
> Once the `"Success"` message appears, all required tables and relationships are built and ready to use.

---

### 3. Host the Application (Render)

Since Securo is split into two separate ecosystems within the same folder structure (Monorepo), you will create **two separate Docker-type Web Services** in Render, both pointing to this same public repository. 

To ensure Render maps the deployment files correctly, fill in the Docker fields and Environment Variables exactly as instructed below:

#### 🏢 A. Configuring the BACKEND (Web Service)

When creating the Backend service, scroll down to the advanced Build settings and define:
* **Dockerfile Path:** `./backend/Dockerfile`
* **Docker Build Context Directory:** `./backend/`

In the **Environment** tab, add the following keys:

| Key | Suggested Value / Explanation |
| :--- | :--- |
| `BACKEND_URL` | The URL generated by Render for this Backend service (e.g., `https://your-backend.onrender.com`) |
| `FRONTEND_URL` | The URL generated by Render for your Frontend service |
| `DATABASE_URL` | Your Supabase database connection string (`postgresql+asyncpg://...`) |
| `SECRET_KEY` | A long, random string of your choice to sign security tokens safely |
| `REDIS_URL` | The internal URL of your own Render Redis instance (see note below 💡) |
| **Pluggy Integration** | |
| `PLUGGY_CLIENT_ID` | The ID generated in your Pluggy developer dashboard |
| `PLUGGY_CLIENT_SECRET` | The Secret generated in your Pluggy developer dashboard |
| **Google Login (OIDC)** | |
| `OIDC_ENABLED` | `true` |
| `OIDC_PROVIDER_NAME` | `Google` |
| `OIDC_DISCOVERY_URL` | `https://accounts.google.com/.well-known/openid-configuration` |
| `OIDC_CLIENT_ID` | Your Client ID generated in the Google Cloud Console (OAuth 2.0) |
| `OIDC_CLIENT_SECRET` | Your Client Secret generated in the Google Cloud Console |
| `OIDC_SCOPES` | `"openid email profile"` |
| `OIDC_AUTO_REGISTER` | `true` |
| `OIDC_REQUIRE_VERIFIED_EMAIL`| `true` |
| `OIDC_EXISTING_USER_LINK_MODE`| `verified_email` |

> 💡 **Note about Redis:** Securo utilizes Redis for caching and managing queues. Before spinning up the backend, click **New +** > **Redis** in the Render dashboard, create a free instance, and copy the **Internal Redis URL** to paste into the backend's `REDIS_URL` field.

---

#### 🎨 B. Configuring the FRONTEND (Web Service)

When creating the Frontend service, adjust the build paths to target the frontend directory:
* **Dockerfile Path:** `./frontend/Dockerfile`
* **Docker Build Context Directory:** `./frontend`

In the **Environment** tab, configure the following routing variables (both are strictly required to ensure the interface can communicate with the API):

| Key | Suggested Value / Explanation |
| :--- | :--- |
| `VITE_API_URL` | The complete URL of your Backend (e.g., `https://your-backend.onrender.com`) |
| `BACKEND_URL` | The exact same complete URL of your Backend |

---

> ⚠️ **Important URL Notice:** Ensure you replace all placeholder addresses with the live URLs generated by Render specifically for your account. Mismatched or misconfigured URLs will cause Google authentication or API calls to fail due to CORS errors.
